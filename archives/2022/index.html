git在配置过公钥后依然需要账号密码的解决方式
github 的新政策让开发者不能通过平常的账号密码来执行 git 操作，转而使用安全性更高的 ssh 密钥来验证 在通过配置好了公钥密钥服务后，施行 git 操作却仍然弹出账号密码的菜单，这主要是因为你所 push 或者 pull 的地址并没有选择 ssh 一般的 ssh 地址为： 1git@github.com:xxxxxx/xxx.git 如果你使用平常的 https 协议，就会要求通过账号密码来登录，从而失败
